# 摄影策划工具 - 安卓端设计方案

## 一、方案概述

| 项目 | 说明 |
|------|------|
| 平台 | Android 原生应用 |
| 最低 SDK | API 26 (Android 8.0) |
| 目标 SDK | API 34 (Android 14) |
| 用户系统 | 无需登录，单用户本地应用 |
| 数据存储 | 全部本地存储（Room 数据库 + 内部文件存储） |
| 网络依赖 | 无，完全离线可用 |

### 设计理念

摄影策划工具的核心场景是：摄影师在电脑/手机上准备参考样图，拍摄现场用手机逐一对照完成。去掉登录和云端存储后：

- **零门槛**：打开即用，无需注册
- **零网络**：拍摄现场无信号也能正常使用
- **零成本**：无服务器、无云存储费用
- **数据安全**：照片不离开手机，隐私有保障

## 二、功能结构设计

### 2.1 首页 —— 策划列表

**功能点：**
- 展示所有已创建的拍摄策划项目
- 每个策划卡片显示：
  - 策划名称
  - 创建时间
  - 完成进度（已完成数量 / 总数，进度条）
  - 封面缩略图（取第一张样图）
- 右下角 FAB 按钮「新建策划」
- 策划卡片支持操作：
  - 点击卡片 → 进入策划详情页
  - 长按 / 更多菜单 → 编辑名称 / 删除策划
- 空状态引导：「点击 + 创建你的第一个拍摄策划」

### 2.2 新建 / 编辑策划页面

**功能点：**
- 输入策划名称
- 图片选择区域：
  - 点击「添加图片」按钮 → 打开系统相册选择器
  - 支持多选图片（系统 Photo Picker）
  - 选择后显示预览网格，可删除单张
- 图片保存到应用内部存储目录
- 保存按钮：保存后跳转到策划详情页
- 编辑模式：可修改名称，可增删图片

### 2.3 策划详情页 —— 拍摄现场主界面

**布局设计：**

```
┌──────────────────────────────┐
│  ← 返回    策划名称    进度 3/8  │  ← 顶部栏
├──────────────────────────────┤
│                              │
│  ┌─────────┐ ┌─────────┐   │
│  │ 未完成1  │ │ 未完成2  │   │  ← 未完成样图
│  └─────────┘ └─────────┘   │    始终在上方
│  ┌─────────┐ ┌─────────┐   │
│  │ 未完成3  │ │ 未完成4  │   │
│  └─────────┘ └─────────┘   │
│  ┌─────────┐               │
│  │ 未完成5  │               │
│  └─────────┘               │
│  ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─  │  ← 分隔线
│  ┌─────────┐ ┌─────────┐   │
│  │ ✓已完成1 │ │ ✓已完成2 │   │  ← 已完成样图
│  └─────────┘ └─────────┘   │    自动沉底
│  ┌─────────┐               │    半透明显示
│  │ ✓已完成3 │               │
│  └─────────┘               │
│                              │
└──────────────────────────────┘
```

**交互逻辑（核心功能）：**

1. 默认状态：所有样图按添加顺序排列，全部「未完成」
2. 点击任意未完成图片 → 勾选「已完成」
   - 该图片自动移动到列表最底部
   - 显示半透明遮罩 + ✓ 完成标记
   - 顶部进度条实时更新
   - 其余未完成图片依次上移
3. 点击已完成图片 → 取消勾选「重新待拍」
   - 图片回到未完成区域顶部
   - 恢复正常显示
4. 点击图片 → 全屏放大查看原图，支持左右滑动切换
5. 所有勾选状态实时保存到本地数据库

### 2.4 图片全屏查看器

- 全屏模态展示
- 左右滑动切换图片
- 双指缩放
- 点击空白区域或返回键关闭

## 三、技术方案

### 3.1 技术栈选型

| 层级 | 技术选型 | 理由 |
|------|----------|------|
| **开发语言** | Kotlin | Android 官方推荐语言，空安全 |
| **UI 框架** | Jetpack Compose | 声明式 UI，开发效率高，动画支持好 |
| **本地数据库** | Room (SQLite) | Jetpack 组件，类型安全，支持 Flow |
| **图片加载** | Coil 3 | Kotlin 优先，Compose 原生支持 |
| **图片选择** | Android Photo Picker (ActivityResultContracts) | 系统原生，无需第三方库 |
| **导航** | Compose Navigation | 类型安全路由 |
| **依赖注入** | 无（手动管理，项目规模小） | 避免过度设计 |
| **异步处理** | Kotlin Coroutines + Flow | 结构化并发，响应式数据流 |

### 3.3 数据模型设计

```kotlin
// 策划实体
@Entity(tableName = "plans")
data class PlanEntity(
    @PrimaryKey(autoGenerate = true) val id: Long = 0,
    val name: String,
    val createdAt: Long,       // 时间戳
    val updatedAt: Long
)

// 样图实体
@Entity(
    tableName = "samples",
    foreignKeys = [ForeignKey(
        entity = PlanEntity::class,
        parentColumns = ["id"],
        childColumns = ["planId"],
        onDelete = ForeignKey.CASCADE
    )]
)
data class SampleEntity(
    @PrimaryKey(autoGenerate = true) val id: Long = 0,
    val planId: Long,
    val localPath: String,       // 应用内部存储路径
    val isCompleted: Boolean = false,
    val sortOrder: Int,          // 添加时的原始顺序
    val createdAt: Long
)
```

### 3.4 本地文件存储方案

```
应用内部存储结构：
/data/data/com.photo.plan/files/
└── images/
    └── {planId}/
        ├── sample_1.jpg
        ├── sample_2.jpg
        └── sample_3.jpg
```

- 使用 `Context.getFilesDir()` 获取内部存储根目录
- 每个策划创建独立子目录
- 图片复制到内部存储，避免依赖外部文件变化
- 删除策划时，级联删除对应目录
- 图片命名规则：`sample_{sortOrder}_{timestamp}.jpg`

### 3.5 排序算法

```kotlin
samples.sortedWith(
    compareBy<SampleEntity> { it.isCompleted }
        .thenBy { it.sortOrder }
)
```

## 四、核心交互流程

### 4.1 用户操作全流程

```
首次使用
  │
  └─→ 打开 App → 首页（策划列表，此时为空）
       │
       └─→ 点击 FAB「新建策划」
            │
            ├─→ 输入策划名称
            ├─→ 点击「添加图片」→ 从相册多选
            ├─→ 预览已选图片，可删除
            └─→ 保存 → 跳转到策划详情页
                          │
                          ├─→ 看到所有样图，全部未完成
                          │
                  拍摄现场
                          │
                          ├─→ 打开 App → 点击目标策划
                          │
                          ├─→ 每拍完一张 → 点击勾选「完成」
                          │    └─→ 图片自动沉底，进度条更新
                          │
                          └─→ 全部拍完 → 所有图片带 ✓ 标记沉底
```

### 4.2 勾选完成的核心逻辑

```
用户点击某张未完成图片
        │
        ▼
前端立即更新本地状态
  1. 将该图片 isCompleted 改为 true
  2. 重新排序列表：未完成在前，已完成在后
  3. 进度条 +1
        │
        ▼
异步写入 Room 数据库
  1. 更新 SampleEntity 的 isCompleted 字段
  2. 无需网络请求，本地事务即可
        │
        ▼
UI 自动响应（Flow 观察）
  - 列表动画过渡
  - 进度条动画更新
```

## 六、与原方案对比

| 维度 | 原方案（Web+云端） | 新方案（安卓+本地） |
|------|---------------------|---------------------|
| 平台 | Web 网页 | Android 原生 |
| 用户系统 | 邮箱+密码注册登录 | 无需登录 |
| 数据存储 | MongoDB + 阿里云 OSS | Room SQLite + 内部文件存储 |
| 网络依赖 | 必须联网 | 完全离线 |
| 跨设备同步 | 支持 | 不支持（可扩展导出功能） |
| 部署成本 | 云服务器+域名+SSL | 零成本 |
| 开发复杂度 | 前后端分离，复杂 | 单端，简洁 |
| 性能 | 受网络影响 | 本地操作，即时响应 |
| 数据安全 | 云端传输有风险 | 数据不离开设备 |
