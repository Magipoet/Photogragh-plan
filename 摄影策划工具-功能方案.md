# 摄影策划工具 - 功能实现方案

## 一、平台选择

**Web 网页应用**，支持响应式设计。

| 特性 | 说明 |
|------|------|
| 电脑端 | 上传和管理样图，新建策划，大屏操作方便 |
| 手机端 | 拍摄现场打开浏览器即可查看，无需安装 App |
| 跨设备同步 | 电脑上传样图 → 云端存储 → 手机实时查看 |

## 二、功能结构设计

### 2.1 首页 —— 策划列表

![首页示意图](https://trae-api-cn.mchost.guru/api/ide/v1/text_to_image?prompt=clean%20modern%20web%20dashboard%20interface%20for%20photography%20planning%20projects,%20showing%20a%20list%20of%20project%20cards%20with%20names,%20dates,%20and%20progress%20indicators,%20minimalist%20design%20with%20white%20background%20and%20subtle%20shadows,%20Chinese%20text%20labels,%20responsive%20layout&image_size=landscape_16_9)

**功能点：**
- 展示所有已创建的拍摄策划项目
- 每个策划卡片显示：
  - 策划名称
  - 创建时间
  - 完成进度（已完成数量 / 总数，进度条）
- 顶部「新建策划」按钮，点击跳转到新建页面
- 策划卡片支持操作：
  - 点击卡片 → 进入策划详情页（拍摄现场使用）
  - 更多菜单 → 编辑 / 删除策划

### 2.2 新建 / 编辑策划页面

**功能点：**
- 输入策划名称（例如："2025 夏季婚纱套餐"、"5.20 工作室写真"）
- 多张图片上传区域：
  - 支持拖拽本地图片到上传区
  - 支持点击选择文件
  - 上传前可预览，可删除单张
  - 上传进度条显示
- 图片上传后自动保存到云端
- 可以随时回到编辑页面添加或删除图片
- 保存按钮：确认后跳转到策划详情页

**交互逻辑（核心功能）：**

1. 默认状态：所有样图按上传顺序排列，全部处于「未完成」状态
2. 点击任意一张未完成图片 → 勾选「已完成」
   - 该图片自动移动到列表最底部
   - 显示半透明遮罩 + ✓ 完成标记
   - 顶部进度条实时更新
   - 其余未完成图片依次上移，填补空位
3. 点击已完成图片 → 取消勾选「重新待拍」
   - 图片回到未完成区域顶部
   - 恢复正常显示，移除标记
4. 点击图片 → 全屏放大查看原图，支持左右滑动切换
5. 所有勾选状态自动保存到云端，下次打开保持不变

### 2.4 用户系统

**极简设计，降低使用门槛：**

- 首次进入 → 输入邮箱 + 密码 → 注册完成
- 后续进入 → 输入邮箱 + 密码 → 登录
- 登录后自动跳转到策划列表首页
- 每个用户只能看到自己创建的策划，数据隔离
- 登录态持久化（关闭浏览器再打开，无需重复登录）

## 三、技术方案

### 3.1 技术栈选型

| 层级 | 技术选型 | 理由 |
|------|----------|------|
| **前端框架** | React 18 + TypeScript | 组件化开发高效，TypeScript 提供类型安全 |
| **样式方案** | Tailwind CSS | 响应式设计方便，手机和电脑自适应 |
| **前端路由** | React Router v6 | 页面跳转管理 |
| **HTTP 请求** | Axios | 和后端 API 通信 |
| **后端框架** | Node.js + Express | 轻量，和前端同语言，开发效率高 |
| **数据库** | MongoDB | 文档型数据库，灵活存储策划和图片数据 |
| **图片存储** | 阿里云 OSS / 腾讯云 COS | 图片文件云端存储，访问快速 |
| **用户认证** | JWT (JSON Web Token) | 无状态认证，简单可靠 |
| **部署** | 单台云服务器 | 前后端部署在同一服务器，成本低 |

### 3.3 数据模型设计

```typescript
// 用户表
interface User {
  _id: string;           // MongoDB 自动生成
  email: string;         // 登录邮箱
  passwordHash: string;  // 加密后的密码
  createdAt: Date;
}

// 策划表
interface Plan {
  _id: string;
  userId: string;        // 关联用户
  name: string;          // 策划名称
  createdAt: Date;
  updatedAt: Date;
}

// 样图表
interface Sample {
  _id: string;
  planId: string;        // 关联策划
  imageUrl: string;      // 云存储图片地址
  thumbnailUrl: string;  // 缩略图地址（列表展示用）
  isCompleted: boolean;  // 是否已完成
  sortOrder: number;     // 排序位置（上传时的原始顺序）
  createdAt: Date;
}
```

### 3.4 API 接口设计

| 方法 | 路径 | 说明 |
|------|------|------|
| `POST` | `/api/auth/register` | 用户注册 |
| `POST` | `/api/auth/login` | 用户登录 |
| `GET` | `/api/plans` | 获取当前用户的所有策划列表 |
| `POST` | `/api/plans` | 新建策划 |
| `GET` | `/api/plans/:id` | 获取单个策划详情（含样图列表） |
| `PUT` | `/api/plans/:id` | 编辑策划名称 |
| `DELETE` | `/api/plans/:id` | 删除策划 |
| `POST` | `/api/plans/:id/samples` | 上传样图到指定策划 |
| `DELETE` | `/api/samples/:id` | 删除单张样图 |
| `PATCH` | `/api/samples/:id/toggle` | 切换样图完成状态 |

## 四、核心交互流程

### 4.1 用户操作全流程

```
首次使用
  │
  ├─→ 打开网页 → 注册账号 → 登录
  │
  └─→ 进入首页（策划列表，此时为空）
       │
       └─→ 点击「新建策划」
            │
            ├─→ 输入策划名称
            ├─→ 拖拽/选择多张参考样图
            ├─→ 等待上传完成
            └─→ 保存 → 跳转到策划详情页
                          │
                          ├─→ 看到所有样图，全部未完成
                          │
                  拍摄现场（手机打开网页）
                          │
                          ├─→ 登录同一账号
                          ├─→ 首页看到所有策划
                          ├─→ 点击目标策划进入详情
                          │
                          ├─→ 每拍完一张样图 → 点击勾选「完成」
                          │    └─→ 图片自动沉底，进度条更新
                          │
                          └─→ 全部拍完 → 所有图片带 ✓ 标记沉底
```

### 4.2 勾选完成的核心逻辑

```
用户点击某张未完成图片的 ✓ 按钮
        │
        ▼
前端立即更新本地状态（乐观更新）
  1. 将该图片 isCompleted 改为 true
  2. 重新排序列表：未完成在前，已完成在后，各自保持 sortOrder
  3. 进度条 +1
        │
        ▼
发送 PATCH /api/samples/:id/toggle 到后端
        │
        ▼
后端更新数据库
  1. 找到该 Sample 记录
  2. 翻转 isCompleted 字段
  3. 返回更新后的数据
        │
        ▼
前端用后端返回数据做最终确认
（若失败则回滚本地状态并提示用户）
```

### 4.3 排序算法说明

样本列表的排序规则：

```
排序优先级：
1. isCompleted === false 的排在前面（未完成的在上面）
2. isCompleted === true  的排在后面（已完成的在下面）
3. 同组内按 sortOrder 升序排列（保持原始上传顺序）

伪代码：
samples.sort((a, b) => {
  if (a.isCompleted !== b.isCompleted) {
    return a.isCompleted ? 1 : -1;  // 未完成优先
  }
  return a.sortOrder - b.sortOrder;  // 同组内按原顺序
});
```

## 五、界面设计要点

### 5.1 响应式断点

| 设备 | 屏幕宽度 | 布局 |
|------|----------|------|
| 手机 | < 768px | 单列网格 / 两列网格 |
| 平板 | 768px - 1024px | 三列网格 |
| 电脑 | > 1024px | 四列或更多列网格 |

### 5.2 关键 UI 细节

- **已完成图片**：半透明黑色遮罩（opacity: 0.4）+ 绿色 ✓ 图标 + 圆角边框变灰
- **未完成图片**：正常显示，边框为浅灰色，悬停时边框高亮
- **进度条**：顶部栏实时显示，动画过渡平滑
- **图片点击**：弹出全屏模态框，支持左右箭头切换、双指缩放（手机）
- **空状态**：新建策划后若无图片，显示引导提示「点击上传参考样图」

## 六、部署方案

### 6.1 部署架构

```
用户浏览器
  ├─ 电脑端 → https://your-domain.com → Nginx 反向代理
  └─ 手机端 → https://your-domain.com → Nginx 反向代理
                                           │
                    ┌──────────────────────┤
                    ▼                      ▼
            前端静态文件             后端 API 服务
         (React build 产物)    (Node.js + Express :3000)
                                           │
                                      ┌────┴────┐
                                      ▼         ▼
                                  MongoDB   云存储 OSS/COS
```

### 6.2 建议服务器配置

| 项目 | 推荐配置 |
|------|----------|
| 云服务器 | 2 核 4G，阿里云 ECS / 腾讯云 CVM |
| 操作系统 | Ubuntu 22.04 / CentOS 8 |
| 域名 | 自行购买，配置 DNS 解析 |
| HTTPS | 使用 Let's Encrypt 免费 SSL 证书 |
| 进程守护 | PM2 管理 Node.js 进程 |

### 6.3 部署步骤概要

1. 购买云服务器，安装 Node.js、Nginx、MongoDB
2. 配置云存储 OSS/COS 并获取 AccessKey
3. 部署后端：上传代码 → npm install → pm2 start
4. 部署前端：npm run build → 将 dist 目录上传到 Nginx 静态目录
5. 配置 Nginx 反向代理，将 `/api` 请求转发到后端端口
6. 绑定域名，配置 HTTPS 证书

## 七、方案特点总结

| 需求 | 满足情况 |
|------|----------|
| 为每次拍摄上传几张样图作为参考 | ✅ 创建多个独立策划，每个策划独立上传样图 |

| 多次策划分别独立管理 | ✅ 首页列出所有策划，互不干扰 |
